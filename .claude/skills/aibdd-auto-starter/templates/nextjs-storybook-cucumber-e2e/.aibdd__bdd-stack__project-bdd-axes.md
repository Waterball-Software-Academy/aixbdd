## §5 Feature 生命週期軸

### §5.1 Feature filename 軸（NORMATIVE — aibdd Step 0）

| axis | 允許 enum | 本專案值 |
|---|---|---|
| `filename.convention` | `NN-prefix-then-title` \| `kebab-all-lower` \| `snake_case` \| `PascalCase` \| `free-form:<regex>` | `NN-prefix-then-title` |
| `filename.title_language` | `zh-Hant` \| `zh-Hans` \| `en` \| `ja` \| `ko` \| `mixed` \| `custom:<iso-code>` | `${PROJECT_SPEC_LANGUAGE}` |

**NN-prefix**：兩位數字 + `-` + 與 `filename.title_language` 一致之標題，例：`01-查詢訂單金額.feature`。

**例外**：`home.feature`（小寫、無數字前綴）僅允許作為 **starter smoke**，路徑鎖定 `${FE_FEATURES_DIR}/home.feature`；**不得**作為 Discovery rule-only 檔名範本。

### §5.2 Red Pre-Hook 軸（NORMATIVE）

| axis | 允許 enum | 本專案值 |
|---|---|---|
| `red.pre_hook` | `none` \| `enabled` | `enabled` |
| `red.pre_hook.ref` | 必與 `arguments.yml` 中 `RED_PREHANDLING_HOOK_REF` 解析後路徑一致 | `.aibdd/bdd-stack/prehandling-before-red-phase.md` |
| `red.pre_hook.fill` | `schema-analysis` \| `composite:openapi+mock+storybook` \| `custom:<label>` \| `none` | `composite:openapi+mock+storybook` |

**綁定**：

- `red.pre_hook = enabled` 時，進入 Red（對目標 `.feature` 第一次去 `@ignore` 並跑 acceptance）之前必須讀取 `red.pre_hook.ref` 所指檔案並完成其 Pre-Red gate。
- nextjs-playwright Pre-Red gate 涵蓋 6 條檢查（OpenAPI / Zod / page.route fixture handler / Storybook binding / Tier-2 enablement / API host separation）；細節見 `prehandling-before-red-phase.md` §3。
- Pre-hook 失敗 → **不啟動** Red 主流程；RGB loop budget **不**視為已消耗於合法紅燈嘗試。
- Hook 內容為 stack-specific；檔名與 §9 鍵維持 stack-neutral。

## §6 Archive / 搬遷（本專案宣告）

| Axis | 允許 enum | 本專案值 |
|---|---|---|
| `feature.archive_mechanism` | `physical-move` \| `build-time-copy` \| `symlink` \| `glob-include` \| `hybrid:<spec>` \| `no-archive` | `symlink` |

**說明**：規格 SSOT 維持在 `${FEATURE_SPECS_DIR}`；runner side 在 `${FE_FEATURES_DIR}/<current_spec_package>/` 用 symlink 鏡像，使單一物理 `.feature` 檔同時是 Discovery / plan 的 truth 與 `bddgen` 的 input。實際 RED 階段 link 步驟、Windows / FAT 環境 last-resort copy、與 visibility check 流程見 `bdd-stack/feature-archive.md`。若未來改機制，須先修訂本節與 `arguments.yml`。
