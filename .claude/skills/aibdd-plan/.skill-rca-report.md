# `/aibdd-plan` Skill RCA

## Defect

Downstream `/aibdd-tasks` needs a stable impacted feature scope, but `/aibdd-plan` originally had no enforced guarantee that the final `plan.md` artifact would contain a valid `## Impacted Feature Files` section.

## Earliest Faulty Artifact

The earliest faulty artifact is the deterministic and semantic quality-enforcement layer, not the plan template itself.

- `assets/templates/plan-md.template.md` can express the section.
- `references/impacted-feature-files-contract.md` defines the contract.
- The missing piece was enforcement in Phase 7 and the semantic veto contract.

## Root Cause

`$$impacted_feature_files` existed only as an in-memory Phase 3 model plus template intent.
There was no deterministic script to read the written `plan.md` and fail when the section was missing, empty, malformed, duplicated, or outside `${FEATURE_SPECS_DIR}`.
The semantic gate also did not veto this gap, so the downstream contract could drift silently.

## Minimal Safe Correction

1. Add `scripts/python/check_impacted_feature_files.py`.
2. Wire that script into Phase 7 deterministic gates in `SKILL.md`.
3. Add an explicit semantic veto to `references/quality-gate-contract.md`.
4. Remove the Phase 10 drift that listed `/aibdd-tasks` as fully out of scope even though `/aibdd-plan` now exposes a downstream task-planning contract.
5. Update fixture quality reports to reflect the extra deterministic gate.

## Risk

Low to medium.
The new gate may fail older or hand-written `plan.md` files that omit or loosely format impacted feature bullets, but that is expected and desirable because those artifacts are not downstream-safe.
