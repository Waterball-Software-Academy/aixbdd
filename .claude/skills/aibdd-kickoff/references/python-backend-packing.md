# Python Backend Packing

本檔定義 `/aibdd-kickoff` 的唯一 Python 後端 top-level boundary packing。Kickoff 只初始化 AIBDD **project config 與 boundary truth skeleton**：`.aibdd/arguments.yml`、architecture truth、唯一 backend boundary root、`shared/dsl.yml`、`test-strategy.yml`、**空的** `packages/` root；**不**建立 constitution、`bdd-stack/*.md`、任何 `packages/NN-<功能模組描述>/`，也不設定 active `TRUTH_FUNCTION_PACKAGE` / `FEATURE_SPECS_DIR` / `ACTIVITIES_DIR`（由 plan-cycle `/aibdd-discovery` 在 sourcing 後 late-bind）；不建立 plan package，也不做 feature-specific 設計。

## Supported Stack

| Dimension | Value | User-facing meaning |
|---|---|---|
| Backend language | Python | 建立 Python 後端 |
| Web framework | FastAPI | 後端 API 入口使用 FastAPI |
| ORM / migration | SQLAlchemy + Alembic | 資料模型與 migration 使用 Python 後端慣例 |
| Test strategy | E2E | BDD 測試會打完整 API 流程 |
| BDD runner | Behave | feature 與 step definitions 使用 Behave |

非 Python / 非 E2E 選項可在說明文字中標示「尚未支援」，但不會進入 artifact render，也不得成為可選選項。

## Unique TLB Model

| Field | Value rule |
|---|---|
| `id` | 使用者確認的 backend service id；建議 kebab-case |
| `level` | `1` |
| `role` | `backend` |
| `type` | `web-service` |
| `description` | `<id> Python FastAPI backend service` 的白話描述 |
| `parent_id` | omitted |

Kickoff 永遠只建立一筆 backend boundary。多 TLB / frontend / external provider 由後續 architecture planning / promotion 能力處理，不在 kickoff 內支援。

## Boundary Root Rule

Kickoff 的唯一 boundary root 永遠是 `${SPECS_ROOT_DIR}/${id}/`。

`type` 只寫入 `boundary.yml` 作為 boundary metadata。Kickoff **不得**自動建立 `${SPECS_ROOT_DIR}/${id}/packages/NN-*`。Functional package 目錄與對應的 `arguments.yml` 鍵只在 `/aibdd-discovery` 綁定後出現。禁止產生 `${SPECS_ROOT_DIR}/${id}/${id}/`、或 `${SPECS_ROOT_DIR}/${id}/${id}/${type}/` 這類多層 root。

## Artifact Pack

| Artifact | Path | Shape |
|---|---|---|
| `arguments.yml` | `${BACKEND_ROOT}/.aibdd/arguments.yml` | `STARTER_VARIANT: python-e2e` + `PROJECT_SPEC_LANGUAGE` + config root + strategy refs + runtime refs + common parameters + Python E2E parameters only |
| `boundary.yml` | `${ARCHITECTURE_DIR}/boundary.yml` | `version: 2` with one backend web-service boundary |
| `component-diagram.class.mmd` | `${ARCHITECTURE_DIR}/component-diagram.class.mmd` | Mermaid `classDiagram` with one `<<web-service>>` class（檔名副檔名見 `aibdd-core::diagram-file-naming.md`） |
| boundary root | `${SPECS_ROOT_DIR}/${id}/` | truth skeleton for the only backend boundary |

注意：上表中的 `${BACKEND_ROOT}` 等於 `${PROJECT_ROOT}/${BACKEND_SUBDIR}`；當 `BACKEND_SUBDIR == ""`（預設），退化為 `${PROJECT_ROOT}`。`${ARCHITECTURE_DIR}` / `${SPECS_ROOT_DIR}` 都是 backend_root 相對的子路徑（見 `convention-mapping.md` §Backend Root 組合規則）。

## Boundary Folder Skeleton

| Folder | Purpose |
|---|---|
| `contracts/` | boundary operation contract root；kickoff writes `contracts/.gitkeep` placeholder. For `web-service`, operation contract files are OpenAPI generated by `/aibdd-form-api-spec` per `aibdd-core::boundary-profile-contract.md` |
| `data/` | boundary state truth root；kickoff writes `data/.gitkeep` placeholder. For `web-service`, state files are DBML generated by `/aibdd-form-entity-spec` per `aibdd-core::boundary-profile-contract.md` |
| `shared/` | boundary-level shared assets；kickoff creates `shared/dsl.yml` placeholder |
| `packages/` | functional package **root only**；kickoff writes `packages/.gitkeep`；**不包含**任一 `packages/NN-*/` |

Kickoff writes `.gitkeep` in `contracts/`, `data/`, and `packages/` so that the empty boundary skeleton survives `git add -A` and fixture filesystem comparison. `.gitkeep` is an empty file. Any non-empty file in these directories is the responsibility of downstream skills (`/aibdd-form-api-spec`, `/aibdd-form-entity-spec`, `/aibdd-discovery`).

Kickoff does not create a boundary-level `actors/` catalog. Discovery derives
Activity actors from the raw idea / impact scope for each plan cycle; an actor
catalog can be introduced later only by a dedicated planning or promotion step.

Kickoff does not create a boundary-level `step-defs/` folder. Runtime step
definitions belong to the automation test tree (for Python E2E, `PY_STEPS_DIR`
under `tests/features/steps`), not to the boundary package skeleton.

Kickoff creates the `contracts/` directory but does not create operation
contract files in it. For the built-in Python backend `web-service` boundary,
the operation contract specifier is `/aibdd-form-api-spec` and the concrete
contract format is OpenAPI. `/aibdd-plan` must delegate API contract formulation
instead of hand-rendering ad hoc operation YAML.

Kickoff creates the `data/` directory but does not create state files in it.
For the built-in Python backend `web-service` boundary, the state specifier is
`/aibdd-form-entity-spec` and the concrete state format is DBML. `/aibdd-plan`
must delegate state/entity formulation instead of hand-rendering YAML data
models.

Kickoff may create placeholder files for `shared/dsl.yml` and `test-strategy.yml`; placeholders must be explicit empty skeletons, not accepted DSL / policy content. `/aibdd-plan` owns semantic DSL content. **`packages/NN-*/dsl.yml` 不得在 kickoff 建立**。

Kickoff must not create `.aibdd/dev-constitution.md`, `.aibdd/bdd-stack/*.md`（含 `project-bdd-axes.md`）；此等專案層 runtime／憲法樹檔案由 `/aibdd-auto-starter` 依 `STARTER_VARIANT` 產生。

## User Confirmation View Requirements

| Item | Plain-language phrasing |
|---|---|
| Supported stack | 本次會建立 Python FastAPI 後端，並用 Behave 跑 E2E BDD 測試 |
| Unique boundary | 本次只建立一個後端服務 boundary，名稱由使用者提供 |
| Architecture files | kickoff 會直接寫入 boundary.yml 與 component-diagram.class.mmd |
| Folder skeleton | kickoff 會建立此後端服務的 contracts / data / shared / packages（root） / test-strategy skeleton；packages 底下的功能模組目錄交由 `/aibdd-discovery` |
| Unsupported options | Java、TypeScript、Frontend-only、Unit Test 尚未支援，不提供選擇 |
