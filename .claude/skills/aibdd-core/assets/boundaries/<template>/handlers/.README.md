# 這個目錄放什麼？

每個 handler 的 runtime（實作期）履約文件，一個 handler 一檔，檔名等於 step-classification.yml 的 handler ID。內容寫該 handler 跑測試時讀寫哪些 context 欄位、何時該選用、禁止做什麼。它描述實作期行為，與談 plan-time 構造的 plugin-contract.md 分工開來。

# 哪裡會用到此目錄下的檔案？

在 .claude/skills/aibdd-red-execute：把 step 依 step-classification.yml 分到某 handler 後，讀這裡對應的 `<handler>.md` 然後交叉比對 `aibdd-core/assets/boundaries/<template>/variants` 之中的 tech stack，來決定如何撰寫測試程式碼才會遵守 "某種handler x 某種技術variants" 的測試撰寫範式。
